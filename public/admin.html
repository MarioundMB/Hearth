Lokales HTTPS

          
        

        

          

            

              

                Passkeys lehnen reine IP-Adressen grundsätzlich ab (WebAuthn-Vorgabe) — das gilt auch mit HTTPS. Du brauchst einen echten Hostnamen. Hearth richtet dafür automatisch mDNS auf dem Server ein: der unten vorgeschlagene .local-Name funktioniert damit im LAN ohne weitere Einrichtung (auf Mac/iOS sofort, unter Windows meist auch).
              

              

                Erstellt ein selbstsigniertes Zertifikat für diesen Hostnamen und startet HTTPS auf einem eigenen Port (4501) neben dem normalen Admin-Port. Der Browser zeigt dabei eine Zertifikatswarnung — normal bei selbstsigniert, kann bestätigt werden. Falls eine Firewall aktiv ist, muss dieser Port zusätzlich freigegeben werden (im LAN reicht das, keine Freigabe nach außen nötig).
              

              

                
                
              

              

              

                

                  Optional: Zertifikat herunterladen und als vertrauenswürdig installieren, damit die Browser-Warnung verschwindet (pro Gerät einmalig — unter macOS: Datei doppelklicken, in Schlüsselbundverwaltung öffnen, dann auf "Immer vertrauen" setzen).
                

                Zertifikat herunterladen
              

            

          

        

      

    


    
    

      

        

          
Admin-Zugriff über 3000 freischalten

          
        

        

          

            

              

                Ein Passkey gilt nur für die Domain, mit der er erstellt wurde — auf dem Gäste-Port (3000) gibt es aber kein Passwort-Login mehr, um dort einen neuen Passkey einzurichten. Der TAP-Key ist ein Einmal-Code (24h gültig, nur für dein Konto), mit dem du auf /setup-access einen Passkey und danach 2FA für die aktuelle Domain einrichten kannst. Der Code verfällt sofort nach erfolgreichem Abschluss.
